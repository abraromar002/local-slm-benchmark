Key findings
⚡
llama3.2:3b is the speed champion
18s avg latency — 5× faster than mistral:7b. Best choice when response time is critical.
🧠
phi4-mini wins on detail and JSON
177 avg words and the only model to pass structured output — best for tasks needing thorough, structured responses.
📊
Bigger model ≠ better on CPU
mistral:7b (4.4GB) is 5× slower than llama3.2 (2GB) with no quality gain — model size hurts on CPU-only inference.
🔥
Cold start penalty is real
TTFT drops from 3.96s (cold) to 0.5s (warm) — an 8× improvement. Always warm up the model before benchmarking.
